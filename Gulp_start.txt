// Что-бы стартонуть новый проэкт 
1. Копируем старт папку
2. Переиминовываем
3. Открываю проэкт в VsCode
4. Открываю терминал в VsCode
5. >npm i
6. Потгружаются все зависимости.
6.2. Для запуска cборщика:
       gulp
       Для остановки:
       Ctrl+C
     Для завершения проэкта в терминале вводим в ручную
     что-бы не нагружать проэкт:
     gulp build
7. В папке html/chunk находятся header, main, footer
   пишим код в них и через html/index.html приобразуем
   в app/index.html который выводится через 
   локальный-сервер.
8. Бургер меню лучше писать сразу по стандартным
    именам классов.( чтобы не исправлять имена в 
    scss и JS )

9. Можно мягко обновить плагины:
    1. проверяем что можно обновить >npm outdated 
    2. если есть что обновлять обновляем >npm update  --save-dev && npm update --save
       обновляет до версии которой не нужно исправлять код ( код остаётся рабочим ).

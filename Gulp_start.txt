// Сначало проэкт на GitHub
   1. Создаём папку проэкта кидаем туда один какой нибудь файл(можно .txt)
   2. С этой папки запускаем VsCode
   3. Открываем Terminal.
   4. Заходим на GitHub.
   5. На сайте GitHub создаём новый проэкт.
   6. Заполняем: 
                 Имя проэкта.
                 Описание проэкта.
                 Выбор видимости: Public
                                  Private
   7. Нажимаем кнопку "Create repository"
   
   8. Заливка на GitHub проэкта:
       С стронички проекта на GitHub с права копируем все сразу команды
       и вставляем их в командную строку все сразу(В ПАПКЕ ПРОЭКТА):
       echo "# Trucking_Service" >> README.md
         git init
         git add README.md
         git commit -m "first commit"
         git branch -M main
         git remote add origin https://github.com/AndreyGalena/Trucking_Service.git
         git push -u origin main
   
    6. Обновляем страничку на GitHub если обновилась то всё ОК.

// Что-бы стартонуть новый проэкт
1. Копируем старт папку
2. Переиминовываем
3. Открываю проэкт в VsCode
4. Открываю терминал в VsCode
5. >npm i
6. Потгружаются все зависимости.
6.2. Для запуска cборщика:
       gulp
       Для остановки:
       Ctrl+C
     Для завершения проэкта в терминале вводим в ручную
     что-бы не нагружать проэкт:
     gulp build
7. В папке html/chunk находятся header, main, footer
   пишим код в них и через html/index.html приобразуем
   в app/index.html который выводится через 
   локальный-сервер.
8. Бургер меню лучше писать сразу по стандартным
    именам классов.( чтобы не исправлять имена в 
    scss и JS )

9. Можно мягко обновить плагины:
    1. проверяем что можно обновить >npm outdated 
    2. если есть что обновлять обновляем >npm update  --save-dev && npm update --save
       обновляет до версии которой не нужно исправлять код ( код остаётся рабочим ).
